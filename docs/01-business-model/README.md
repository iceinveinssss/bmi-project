### 1.2 Бизнес-контекст

**Проблема:** Большинство пользователей не имеют удобного инструмента для регулярного мониторинга индекса массы тела. Существующие решения либо не сохраняют историю измерений, либо требуют постоянного подключения к интернету, либо не предоставляют персонализированную аналитику.

**Решение:** Мобильное приложение «ИМТ Калькулятор» позволяет пользователям рассчитывать ИМТ по весу и росту, сохранять каждое измерение на сервере, просматривать историю и статистику, а также работать в режиме офлайн с помощью локального кэширования (AsyncStorage, ключи `bmi_history_cache_v1` и `bmi_stats_cache_v1`).

### 1.3 Цели проекта

- Разработать кроссплатформенное мобильное приложение (iOS и Android) для расчёта и мониторинга ИМТ.
- Реализовать серверную часть на Java 17 + Spring Boot 3.2.0 с REST API, JWT-аутентификацией и документацией OpenAPI.
- Обеспечить оффлайн-режим работы с кэшированием данных.
- Реализовать архитектурный паттерн PCMEF на сервере и адаптированный вариант на клиенте.

## 4. Бизнес-глоссарий (20 терминов)

| Термин | Определение |
|---|---|
| ИМТ (индекс массы тела) | Показатель: масса (кг) / (рост (м))². Реализован как статический бизнес-метод `BmiRecord.calculateBmi(weightKg, heightCm)` |
| Категория ИМТ | Классификация ВОЗ из 7 градаций; определяется методом `BmiRecord.getCategory(bmi)` |
| Запись ИМТ (`BmiRecord`) | JPA-сущность: id, weight, height, bmi, category, measuredAt, user |
| Пользователь (`User`) | JPA-сущность: id, email, password, name, birthYear, role, createdAt |
| Роль (`User.Role`) | Перечисление: `USER` (по умолчанию при регистрации) или `ADMIN` |
| JWT | Токен с claims: subject (email), userId, role. Срок жизни — 86 400 000 мс (24 ч) |
| BCrypt | Алгоритм хеширования паролей (`BCryptPasswordEncoder`) |
| `authStorage` | Модуль хранения токена и профиля: `expo-secure-store` на iOS/Android, `AsyncStorage` на web |
| `SecureStore` | `expo-secure-store` — зашифрованное хранилище токена и данных пользователя на мобильном устройстве |
| `AsyncStorage` | Локальное хранилище кэша истории (`bmi_history_cache_v1`) и статистики (`bmi_stats_cache_v1`) |
| Оффлайн-режим | Режим работы при недоступности сервера; данные берутся из кэша AsyncStorage |
| REST API | Архитектурный стиль взаимодействия клиент–сервер через HTTP/JSON |
| OpenAPI / Swagger UI | Интерактивная документация API: `http://localhost:8082/swagger-ui.html` |
| PCMEF | Presentation–Control–Mediator–Entity–Foundation — архитектурный паттерн проекта |
| Spring Security | Stateless-конфигурация: JWT-фильтр, CSRF отключён, сессии не используются |
| `JwtFilter` | `OncePerRequestFilter`: извлекает email, userId, role из токена и устанавливает `SecurityContext` |
| Lombok | Библиотека генерации кода: `@Getter`, `@Setter`, `@Builder`, `@RequiredArgsConstructor` |
| JaCoCo | Плагин Maven (0.8.11) для измерения покрытия кода тестами |
| Метрические единицы | Вес — кг, рост — см (хранятся в БД всегда в метрических единицах) |
| Имперские единицы | Вес — lb, рост — in; конвертация через функции `units.js` на клиенте перед отправкой |

### Таблица категорий ИМТ

| Диапазон ИМТ | Категория (`BmiRecord.getCategory()`) |
|---|---|
| < 16.0 | Выраженный дефицит массы |
| 16.0 – 18.4 | Недостаточная масса тела |
| 18.5 – 24.9 | Норма |
| 25.0 – 29.9 | Избыточная масса тела |
| 30.0 – 34.9 | Ожирение I степени |
| 35.0 – 39.9 | Ожирение II степени |
| ≥ 40.0 | Ожирение III степени |

## 7. SWOT-анализ

| | Возможности | Угрозы |
|---|---|---|
| **Сильные стороны** | Расширить до health-трекера; интеграция с Google Fit / Apple Health | Открытый стек снижает зависимость от платформ |
| **Слабые стороны** | Добавить push-уведомления, графики динамики ИМТ | Конкуренция со стороны крупных health-платформ |

**Сильные стороны:** кроссплатформенность (iOS + Android через Expo), оффлайн-режим, JWT-безопасность, 7-ступенчатая классификация ВОЗ, поддержка метрических и имперских единиц, тёмная и светлая тема.

**Слабые стороны:** нет push-уведомлений, нет синхронизации с фитнес-трекерами, нет визуальных графиков динамики.

**Возможности:** интеграция с Google Fit / Apple Health, публикация в Google Play / App Store, добавление графиков динамики ИМТ.

**Угрозы:** конкуренция с MyFitnessPal и аналогами; требования к защите персональных медицинских данных.
